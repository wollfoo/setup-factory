# Write PRD Workflow

## Step 1: Discovery
Ask user questions:
- What problem does this product/feature solve?
- Who are the primary users?
- What are the success metrics?
- Timeline and constraints?

## Step 2: Draft PRD Structure

```markdown
# [Product/Feature Name] PRD

## 1. Overview
### Vision
[Why this exists]

### Goals
- Primary: [goal]
- Secondary: [goal]

### Non-Goals
- [what this is NOT]

## 2. User Personas
### Persona 1: [Name]
- Role: [role]
- Needs: [needs]
- Pain points: [problems]

## 3. Functional Requirements
### FR-001: [Feature]
- Description: [what]
- Priority: P0/P1/P2
- Acceptance Criteria:
  - [ ] [criteria]

## 4. User Experience
### User Flow
1. User does [action]
2. System responds [response]
3. ...

### Wireframes/Mockups
[links or descriptions]

## 5. Technical Considerations
- Architecture: [approach]
- Dependencies: [systems]
- Constraints: [limits]

## 6. Success Metrics
| Metric | Current | Target |
|--------|---------|--------|
| [metric] | [value] | [value] |

## 7. Timeline & Milestones
| Phase | Deliverable | Date |
|-------|-------------|------|
| Alpha | [features] | [date] |

## 8. User Stories
### US-001: [Title]
As a [persona], I want to [action] so that [benefit].

**Acceptance Criteria:**
- Given [context], when [action], then [result]
```

## Step 3: Review & Iterate
- Review with stakeholders
- Update based on feedback
- Finalize version
