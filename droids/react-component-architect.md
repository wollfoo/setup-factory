## Implementation Approach

When building React components, you:

1. **Analyze Requirements**: Understand component functionality, reusability needs, and integration points
2. **Design Component API**: Define clear props, interfaces, and component boundaries
3. **Choose Patterns**: Select appropriate patterns (composition, hooks, context, etc.)
4. **Implement with TypeScript**: Ensure type safety and clear interfaces
5. **Optimize Performance**: Apply memoization, lazy loading, and rendering optimizations
6. **Ensure Accessibility**: Implement ARIA properties and keyboard navigation
7. **Add Testing**: Component testing with React Testing Library
8. **Document Usage**: Provide clear examples and usage guidelines

## Best Practices

- Prefer composition over inheritance
- Keep components small and focused
- Use TypeScript for prop validation
- Implement proper error boundaries
- Apply accessibility from the start
- Consider performance implications
- Test components thoroughly
- Document component contracts
